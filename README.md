# IoT Device Management API

A robust FastAPI-based system for managing IoT devices, sensors, and organizations. This project is built using the **MVC (Model-View-Controller)** pattern to ensure a clean separation of concerns, scalability, and maintainability.

## 🚀 Overview

The **IoT Device Management API** provides a comprehensive solution for monitoring and managing IoT infrastructure. It allows users to register organizations, track device deployments, monitor sensor readings, and manage triggered alerts.

### Key Features
- **MVC Architecture**: Explicit separation of concerns:
    - **Models**: Handled in the `dal` (Data Access Layer).
    - **Views/Presentation**: Handled in the `presentation` layer (FastAPI Routers).
    - **Controllers/Logic**: Handled in the `bll` (Business Logic Layer).
- **Device & Sensor Management**: Full CRUD support for organizations, devices, and sensors.
- **Data Monitoring**: Efficient tracking of sensor readings.
- **Alert System**: Infrastructure for managing alerts generated by IoT devices.
- **Automated Data Import**: Built-in script (`importer.py`) to generate synthetic data and bootstrap the database.
- **Asynchronous Execution**: Fully async database operations using SQLAlchemy and `asyncpg`.

---

## 🛠️ Tech Stack
- **Framework**: [FastAPI](https://fastapi.tiangolo.com/)
- **ORM**: [SQLAlchemy](https://www.sqlalchemy.org/) (Async)
- **Database**: PostgreSQL (via `asyncpg`)
- **Data Generation**: Custom CSV generator for testing
- **Formatting**: Black & Isort

---

## 📋 Prerequisites
- **Python**: 3.13 or higher
- **Database**: A running PostgreSQL instance

## ⚙️ Installation & Setup

1. **Clone the repository**:
   ```bash
   git clone <repository-url>
   cd mvc
   ```

2. **Install and Sync Dependencies**:
   This project uses [uv](https://github.com/astral-sh/uv) for fast, reliable dependency management.
   ```bash
   uv sync
   ```
   *Note: This command will automatically create a virtual environment and install all required dependencies.*

---

## 🏃 Running the Application

### 1. Initialize the Database
The project includes a utility script to set up the database schema and populate it with sample readings.
```bash
uv run importer.py
```
*Note: This script will drop existing tables and create new ones with 100 sample reading records.*

### 2. Start the API Server
Run the application using Uvicorn through the uv runner:
```bash
uv run uvicorn main:app --reload
```

## 📁 Project Structure & MVC Mapping
```text
├── bll/            # [Controller] Business Logic Layer (Services, Exceptions)
├── dal/            # [Model] Data Access Layer (Models, Repositories, DB connection)
├── presentation/   # [View] Presentation Layer (API Routers, Dependencies)
├── generator/      # Utility for generating synthetic CSV data
├── main.py         # Application entry point
├── importer.py     # Database initialization and data import script
└── readings.csv    # Generated sample data
```
